Circuito implementado para la toma de datos

Resumen


¿Que requiere un sistema de adquisición de datos? ¿Es muy complejo implementarlo? estas y otro tipo de preguntas se resuelven en este proyecto, el cual explica paso a paso como se conforma un prototipo partiendo del hardware hasta el software, dando una explicación completa de lo que se requiere en cada sección, el porque se hace de esta forma en particular y la complejidad que pueda llegar a tener un proyecto de esta índole, proponiendo una solución eficiente a la forma de tomar, organizar y administrar la cantidad de datos que se pueden tomar. Por otra parte, esta solución tendrá inconvenientes que se plantearan con detenimiento, ofreciendo soluciones que puedan sobrellevar las situaciones que se pueden dar sin perder de vista los objetivos que se plantean para el proyecto en cuestión.


Introducción


Cada vez que un físico experimental va a tomar datos o a realizar algo relacionado con el tratamiento de datos, se enfrenta a tres problemas: tomar los datos, guardarlos y luego hacer el respectivo tratamiento. Estos problemas se pueden sobrellevar si no son demasiados datos o si se cuenta con algún tipo de apoyo, pero ¿es lo más eficiente? La respuesta es no. Lo más eficiente siempre será tomar los datos de forma electrónica y guardarlos en un sistema cuya organización este acorde a las necesidades. de organización consuma tiempo, este mismo proceso será necesario en etapas posteriores de la investigación.
Aunque la problemática es bastante clara, generalmente la solución no es tan concreta y, en algunos casos, puede ser más compleja que el problema que se pretende resolver. Para ello, es necesario tener en cuenta algunos factores a la hora de decidir implementar un sistema de adquisición de datos. Por ejemplo, si se requiere tomar un tipo de dato 10 o 20 veces, se puede tomar con papel y lápiz y no habrá mayor complicación. Sin embargo, al implementar un sistema de adquisición de datos, se debe tener en cuenta qué se quiere medir, cómo se desea medir y cómo se desea almacenar los datos. Ahora bien, si se trata de tomar 80 muestras de tres medidas con la fecha y hora en que se tomaron, puede llegar a ser conveniente la implementación de un sistema, ya que es relativamente sencilla la organización de los datos, donde en una tabla con los respectivos campos que se desean medir y un campo que sea representativo de cada muestra, para relacionar las diferentes muestras con los datos de la misma. A su vez, un sistema de adquisición de datos tiene ventajas muy claras sobre los métodos tradicionales. Una vez implementado correctamente, puede tomar todos los datos que se requieran hasta que se acabe el espacio de la base de datos.De esta forma, se tiene un sistema bastante eficiente para administrar los datos, lo que implica la posibilidad de corregir y cambiar sin perder el orden. Por lo tanto, es una solución que se puede ampliar tanto como se necesite. Sin embargo, se debe tener presente el objetivo fundamental del problema, ya que modificar el sistema una vez implementado puede llegar a ser contraproducente dependiendo del tipo de cambio que se desea realizar. Por ello, en este proyecto se intentará proporcionar ciertas pautas con respecto a la construcción del prototipo que ayuden a identificar lo que se debe exigir a la hora de implementarlo.
